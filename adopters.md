## How to add your library

1. Read [SPEC.md](SPEC.md).
2. Open a PR against this repo adding a row to the status table above with current adoption state per section.
3. File tracking issues against your library's repo for each unimplemented section. Link them back to this contract repo in the issue body.
4. Cross-link this repo from your library's README so consumers can find the contract.
5. Update the row as sections land in subsequent releases.
6. **Enumerate precalculated tables** per SPEC [§8.0 catch loop](SPEC.md#80-bit-allocation-for-lib_x_shared_primitives). Required for every precalculated table that meets the §8.0 floor (≥ 256 B AND one of: REU-resident, hot-loop-read, page-aligned for fetch alignment). Two forms, both mandatory:
   - **`docs/precalc-tables.md`** in your library, one row per table: name, size, region (main RAM / REU bank / RODATA), source file, classification (curve-/algorithm-specific *or* potentially shareable), and the rationale for the classification.
   - **`LIB_PRECALC_TABLE` macro invocations** from one of your library's translation units. Copy `precalc_table.inc` verbatim from SPEC §8.0 into `src/precalc_table.inc`, `.include` it once, and emit one `LIB_PRECALC_TABLE "name", size, region, shared` per enumerated table. The macro exports three equates per table (`LIB_PRECALC_<NAME>_{SIZE,REGION,SHARED}`) that cross-adopter audits grep.

   Link `docs/precalc-tables.md` from your row in the status table above.

   **Intake reviewers MUST confirm step 6 is satisfied before merging the adopter PR.** A `LIB_PRECALC_*` export without a corresponding `docs/precalc-tables.md` row — or vice versa — blocks merge. A library that has no tables clearing the §8.0 floor still files `docs/precalc-tables.md` with a one-line "no tables above the floor" note and the rationale (e.g., "all internal scratch is single-byte ZP; rotation lookups are 16 B").
